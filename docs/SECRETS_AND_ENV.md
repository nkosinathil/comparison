# Secrets and Environment Files

This document explains which files in this repository are templates (safe to commit) and which are local-only (must never be committed).

---

## Quick reference

| File | Status | Contains secrets? | Action |
|------|--------|-------------------|--------|
| `deploy.conf.example` | ✅ Committed template | No — placeholders only | Copy to `deploy.conf` and fill in |
| `deploy.conf` | 🔒 Local only (gitignored) | Yes — real secrets | Never commit |
| `deploy/scripts/deploy.conf.example` | ✅ Committed template | No — placeholders only | Same as root example |
| `deploy/scripts/deploy.conf` | 🔒 Local only (gitignored) | Yes — real secrets | Never commit |
| `php-app/.env.example` | ✅ Committed template | No — placeholders only | Copy to `php-app/.env` and fill in |
| `php-app/.env` | 🔒 Local only (gitignored) | Yes — real secrets | Never commit |
| `python-backend/.env.example` | ✅ Committed template | No — placeholders only | Copy to `python-backend/.env` and fill in |
| `python-backend/.env` | 🔒 Local only (gitignored) | Yes — real secrets | Never commit |

## deploy.conf (root)

**Template:** `deploy.conf.example`  
**Live file:** `deploy.conf` (gitignored)

This is the central deployment configuration used by all deployment scripts. It contains:

- Server IPs and SSH usernames
- Keycloak admin credentials
- Database credentials
- MinIO credentials
- The shared PHP↔Python API key
- The branch/tag to deploy (`DEPLOY_REF`)

### How to create yours

```bash
cp deploy.conf.example deploy.conf
nano deploy.conf   # replace all __CHANGE_ME__ values
```

The `deploy-all.sh` script will find `deploy.conf` at the repo root automatically.  
The `deploy-windows.cmd` script reads values directly from Windows environment variables.

## PHP app environment: php-app/.env

**Template:** `php-app/.env.example`  
**Live file:** `php-app/.env` (gitignored — also excluded from rsync during deploy)

In production, this file is written automatically by `setup-app-server.sh` using values from `deploy.conf`. You do not need to create it manually on the server.

Key secrets in this file:

| Variable | Description |
|----------|-------------|
| `KEYCLOAK_CLIENT_SECRET` | Auto-generated by Keycloak setup — do not share |
| `DB_PASSWORD` | PostgreSQL password for `comparison_user` |
| `PYTHON_API_KEY` | Must match `API_KEY` in the Python backend `.env` |

### Callback URI

The `KEYCLOAK_REDIRECT_URI` must be:

```
http://file.gismartanalytics.com/auth/callback
```

(No `.php` extension — the router dispatches it via `index.php`.)

### TLS and JWT flags

```ini
# Set to false for LAN/HTTP dev only. Must be true in production.
KEYCLOAK_TLS_VERIFY=true

# Set to true ONLY in non-production, non-public environments.
# Allows JWT decoding without signature verification when JWKS is unreachable.
# NEVER enable in production.
KEYCLOAK_ALLOW_INSECURE_JWT_DEV=false
```

## Python backend environment: python-backend/.env

**Template:** `python-backend/.env.example`  
**Live file:** `python-backend/.env` (gitignored)

In production, this file is written automatically by `setup-python-server.sh` from `deploy.conf`.

Key secrets:

| Variable | Description |
|----------|-------------|
| `API_KEY` | Must match `PYTHON_API_KEY` in PHP `.env` |
| `MINIO_ACCESS_KEY` | MinIO access credential |
| `MINIO_SECRET_KEY` | MinIO secret credential |

## What is safe to commit

Only files with `__CHANGE_ME__`, `your_*_here`, or `CHANGE_ME_TO_*` placeholder values are safe. Files with real passwords, tokens, or secrets must never be committed.

### Generating secrets locally

```bash
# API key
python3 -c "import secrets; print(secrets.token_urlsafe(32))"

# Database password (strong random)
python3 -c "import secrets, string; print(''.join(secrets.choice(string.ascii_letters + string.digits + '!@#') for _ in range(24)))"
```

## .gitignore entries

The following patterns prevent accidental commits:

```gitignore
.env
deploy.conf
deploy/scripts/deploy.conf
```

If you accidentally commit a secret, rotate it immediately and use `git filter-repo` or BFG Repo Cleaner to purge the history.
